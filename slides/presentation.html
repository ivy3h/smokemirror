System Overview

Smokemirror generates dual-narrative crime mysteries where the reader knows the truth but watches a detective get systematically misled by conspirators.

What the Reader Sees

  • The real crime and who committed it
  • Conspirators planting false evidence
  • Fabricated alibis being constructed
  • Each intervention that blocks the detective
  • Dramatic irony as truth remains hidden

What the Detective Sees

  • A seemingly straightforward crime
  • Evidence pointing to a false suspect
  • Alibis that appear to check out
  • "Helpful" witnesses (who are conspirators)
  • A case that seems to close neatly

Crime Story Design: Non-Trivial Mysteries

Our system addresses all core crime mystery elements:

Multiple Suspects with Means, Motive, Opportunity

2-4 conspirators, each with distinct roles, motives, and leverage. A fake suspect is fabricated with planted evidence.

Red Herrings & False Alibis

A complete fabricated narrative is generated, including planted physical evidence, forged documents, and coordinated alibis.

Multi-Step Clues

Evidence requires 2-3 investigation steps to fully obtain. Physical/digital evidence needs more steps than testimonial.

Active Criminal Obstruction

Conspirators actively intervene when the detective approaches the truth, redirecting the investigation and closing discovery paths.

Suspense Theory: Four Core Elements

Based on Gerrig's theory of suspense and the iterative planning approach (Xie et al., 2024):

(a)
Reader Affinity

Detective with detailed personal background, personality, and stakes the reader can care about

(b)

personal_stakes field

(c)
Dire Consequence

Specific serious outcome if the detective fails (career ruin, innocent convicted)

dire_consequence field

(d)

Countdown mechanism + path closures make success increasingly unlikely each turn

time_remaining, success_probability

LLM and Non-LLM Components

LLM Components

  • Crime Backstory Generator — Creates criminal, victim, conspirators, evidence, timeline
  • Fabricated Narrative Generator — Invents false suspect, plants evidence, creates alibis
  • Detective Profile Generator — Builds protagonist with personal stakes
  • Detective Action Generator — Proposes next investigative steps
  • Conspirator Intervention Generator — Creates misdirection responses
  • Story Assembler — Converts plot points to prose
  • Reader Simulator — 3 reader types evaluate the story

Model: Qwen3-8B (4-bit quantized), supports thinking mode

Non-Neural Components

  • CollisionDetector — Keyword-based semantic matching to detect when detective approaches truth
  • SuspenseMetaController — External control loop managing the entire generation process
  • StoryState Manager — Tracks countdown, paths, probability, evidence progress
  • Investigation Agenda Builder — Structures pre-generated facts into actionable leads
  • Feedback Aggregator — Synthesizes reader feedback, detects consensus issues
  • MetricsCalculator — Computes suspense curves, quality scores (0-100)

Pure Python logic, no neural inference

The core external control loop that walks the LLM through suspenseful story creation:

1

Generate Detective Action

LLM proposes next action given full accumulated history of ALL previous actions and outcomes

2

Collision Detection

Non-neural keyword matching checks if action threatens to expose truth

3

Branch: Intervention or Progress

If collision: conspirator intervenes. Otherwise: standard obstacle or evidence discovery

4

Update State

Tick countdown, close paths, decrease success probability, update evidence progress

Collision Detection: The Non-Neural Guard

A pure keyword-based module that detects when the detective's action could expose the conspiracy:

Four Detection Layers

1. Conspirator Investigation
Partial name matching + investigation verbs (interview, question, verify, confront...)

2. Evidence Examination
Keyword overlap (2+) between action and evidence descriptions

3. Location Visits
Checks if detective visits a crime-relevant location from the timeline

4. Discovery Path Proximity
Keyword overlap with open paths + character name matching

How It Creates Suspense

When a collision is detected:

  • A conspirator actively intervenes
  • The discovery path is closed
  • Success probability decreases 6-12%
  • Suspense level increases
  • Detective is redirected toward the fabricated suspect
  • The countdown keeps ticking — time is wasted

Sensitivity parameter (default 0.5) controls how often collisions trigger, preventing every action from being blocked.

Addressing Template-Specific Questions

Question: How do you adapt the suspense technique to use pre-generated crime details?

Our Approach: Investigation Agenda

Crime details are pre-generated before the iterative loop begins. We bridge the gap with:

  1. Investigation Agenda Builder — converts pre-generated facts into a structured "menu" of evidence, alibis, and suspects for the LLM
  2. Multi-step evidence tracking — each evidence item has specific required investigation steps
  3. Collision Detector — uses keyword matching against pre-generated facts to trigger interventions
  4. Fabricated Narrative validation — ensures planted evidence is consistent with and mirrors real evidence

How Crime Details Stay Consistent

  • Facts generated as a single coherent JSON structure with cross-references
  • Fabricated narrative is validated against real facts for consistency
  • Each turn, the LLM sees the full accumulated history — preventing contradictions
  • Evidence has dual meanings: real_meaning (truth) and fabricated_meaning (what detective sees)
  • Timeline events are pre-coordinated between criminal and conspirators

Pitfalls Encountered & Solutions

Pitfall: LLM Falling Back on Tropes

Problem: LLM generates generic "the detective had a hunch" instead of specific investigative actions.

Solution: Investigation Agenda provides a concrete menu of evidence items, alibis, and locations. The LLM must reference specific pre-generated elements.

Pitfall: Suspense Flatness

Problem: Suspense level stayed constant through the middle of the story.

Solution: Countdown mechanism + decreasing success probability ensures natural escalation. Conspirator interventions spike suspense. Quality flags detect flat middles.

Pitfall: Detective Solving Too Easily

Problem: The detective would sometimes stumble directly onto the truth.

Solution: CollisionDetector with tunable sensitivity (0.5 default). When truth is approached, conspirators intervene and redirect. Discovery paths close systematically.

Pitfall: Disconnected Crime Details

Problem: Pre-generated facts not deeply integrated into the iterative loop.

Solution: Investigation Agenda + multi-step evidence tracking + alibi verification system. Every turn references specific pre-generated evidence IDs and character names.

Identified Problems:

  • Layer leak: The Logic Analyst reader correctly predicted the real criminal due to a too-obvious conspirator alibi
  • Flat middle: Suspense level was constant (4-5) for plot points 4-8 with no escalation
  • Too few plot points: Discovery paths closed too quickly (all closed by point 11), forcing premature resolution
  • Premature peak: Suspense peaked at plot point 3, then declined
  • Detective actions became repetitive — "re-examine evidence" appeared 3 times

Root Causes & Analysis

Layer Leak: The conspirator's alibi was too weakly constructed. The fabricated narrative didn't have enough planted evidence to be convincing. The reader simulation correctly flagged that "the secretary's claim to find the body exactly when expected is suspicious."

Flat Middle: Collision sensitivity was too low for this run. No conspirator interventions occurred between plot points 4-8, so there were no tension spikes. The countdown wasn't urgent enough to drive escalation.

Premature Path Closure: The system closed 4/5 paths in the first 5 plot points due to random path closure probability. This left insufficient paths for the middle and end of the story.

Feedback Aggregation

  • Priority 1: Layer leaks — Must be fixed immediately
  • Priority 2: Suspense curve issues — Flat middles, drops
  • Priority 3: Logical inconsistencies
  • Priority 4: Pacing & engagement issues
  • Consensus threshold: 2+ readers must agree on an issue

Revision Process

  • Up to 3 refinement iterations
  • Only flagged plot points are revised (not entire story)
  • Each revision sees surrounding context (prev/next plot points)
  • Revised points get suspense level +1 boost
  • Re-evaluation after each revision pass
